# 项目启动指南

## 项目结构
- **后端**：Django + Django REST Framework + JWT认证
- **前端**：Vue 3 + Vite + Element Plus + Pinia + Vue Router

## 启动步骤

### 1. 后端启动

#### 1.1 安装依赖
```bash
# 进入后端目录
cd backend

# 安装Django和相关依赖
pip install django django-rest-framework rest-framework-simplejwt corsheaders
```

#### 1.2 初始化数据库
```bash
# 执行数据库迁移
python manage.py makemigrations
python manage.py migrate

# 创建超级用户（可选，用于后台管理）
python manage.py createsuperuser
```

#### 1.3 启动后端服务
```bash
# 启动开发服务器
python manage.py runserver
```

后端服务将运行在 `http://localhost:8000`

### 2. 前端启动

#### 2.1 安装依赖
```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install
```

#### 2.2 启动前端开发服务器
```bash
# 启动Vite开发服务器
npm run dev
```

前端服务将运行在 `http://localhost:5173`（默认端口，可能会自动调整）

## 访问项目

1. 前端应用：打开浏览器访问 `http://localhost:5173`
2. 后端API：`http://localhost:8000/api/`
3. Django后台管理：`http://localhost:8000/admin/`（需要超级用户账号）

## 开发注意事项

1. **跨域配置**：后端已配置CORS允许所有来源，开发环境无需额外配置
2. **数据库**：开发环境使用SQLite，无需额外安装数据库
3. **JWT认证**：前端请求需要在Header中携带JWT Token
4. **API调试**：可以使用Postman或类似工具测试后端API

## 常用命令

### 后端
- `python manage.py runserver` - 启动开发服务器
- `python manage.py makemigrations` - 生成数据库迁移文件
- `python manage.py migrate` - 执行数据库迁移
- `python manage.py createsuperuser` - 创建超级用户
- `python manage.py shell` - 启动Django shell

### 前端
- `npm run dev` - 启动开发服务器
- `npm run build` - 构建生产版本
- `npm run preview` - 预览生产构建

## 技术栈

### 后端
- **Django**：Web框架
- **Django REST Framework**：REST API框架
- **JWT**：身份认证
- **CORS Headers**：跨域支持

### 前端
- **Vue 3**：前端框架
- **Vite**：构建工具
- **Element Plus**：UI组件库
- **Pinia**：状态管理
- **Vue Router**：路由管理
- **Axios**：HTTP客户端

## 项目功能

- 用户认证（登录、注册、个人资料）
- Bug管理（创建、查看、编辑、删除）
- 模块管理
- 用户管理

## API文档

后端API采用RESTful设计，主要端点包括：
- `/api/users/` - 用户管理
- `/api/bugs/` - Bug管理
- `/api/modules/` - 模块管理
- `/api/token/` - JWT令牌获取
- `/api/token/refresh/` - JWT令牌刷新

更多API详情可通过Django REST Framework的浏览界面查看：`http://localhost:8000/api/`
